# Contributing

Use synthetic data only. Never commit documents, secrets, database files, exports, logs, or screenshots containing real client information.

1. Create a focused branch.
2. Run `npm install` and `npm run db:setup`.
3. Keep domain rules out of React components and preserve server-side role checks.
4. Add meaningful tests for rule or workflow changes.
5. Run `npm run lint`, `npm run typecheck`, `npm run test`, and `npm run build`.
6. Describe privacy, accessibility, authorization, audit, and human-review effects in the pull request.

User-facing language must not imply eligibility determination, client ranking, legal authority, or authoritative AI extraction. Report security concerns privately rather than opening a public issue with sensitive detail.
